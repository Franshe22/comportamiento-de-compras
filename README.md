## 📊 Instacart Customer Shopping Analysis
## 📝 Project Description

This project analyzes customer purchasing behavior using a modified dataset from Instacart, an online grocery delivery platform similar to Uber Eats and DoorDash.

The dataset was originally released in 2017 for a competition on Kaggle. For this project, the dataset was modified by reducing its size and introducing missing values and duplicates to simulate real-world data challenges.

The main goal of the project is to clean and explore the data to uncover insights about customer shopping habits, such as purchasing frequency, popular products, and ordering patterns.

## 🎯 Project Objective

The objective of this project is to:

Clean and preprocess the dataset by identifying missing values and duplicates

Perform Exploratory Data Analysis (EDA) to understand customer purchasing patterns

Create data visualizations to communicate key insights

Generate a data-driven report explaining the results

This project simulates a real-world data analyst workflow, from raw data cleaning to insight generation.

## 📈 Key Insights

The analysis helps identify patterns in how customers interact with the Instacart platform, including:

Peak ordering hours and days

Most frequently purchased products

Customer reordering behavior

General shopping habits in online grocery platforms

These insights can help businesses optimize inventory, marketing strategies, and delivery operations.
